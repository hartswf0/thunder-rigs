# PROGRAM THEORY — one surface, film as game

The failure mode this theory forbids: **parallel universes.** The production
board, separate loading toasts, bot-based casting — each was a second way to do
something the engine already did. Bolted-on features feel loose because they
*are* loose: they bypass the operational surface the player already learned.

## Law 1 — ONE OPERATIONAL SURFACE

Every AI act, everywhere in the program, runs through the same four organs:

| organ | what it is | where |
|---|---|---|
| forge weather | `aiThinking` — sun dims, teal sparks while the model works | frame loop |
| build drone | `SFX.buildStart/Stop` — generation is audible | SFX |
| AI·LOG | `AICON.log` — every request, result, failure, in one ledger | AICON |
| the ritual | HELLO/WORLD editors — the ONLY body/world editors | ritual |

A film asset forging must look, sound, and log exactly like a rig forging,
because it IS the same act. Casting = the HELLO generator (never bots).
Set pieces = the fort forge. Editing a character = the HELLO ritual on that
character's rig. No feature may introduce a second loading state, a second
editor, or a second log.

## Law 2 — A FILM IS A GAME

A film is not a mode beside the game; it is a **cartridge** (`rules`,
`entities`, `cine`) that you *load through PLAY* like any game package:

- composing = authoring a cartridge (cast → entities, set → world/forts,
  shots → cine, beats → rules of the performance)
- performing = playing it (drive a character or the camera; the clock is the
  level)
- exporting = the take; saving = a `.trig.json`/`.twin.json` on the shelf

Next concrete step: composed films register in the PLAY games list, so
"load a film" and "load a game" are the same gesture.

## Law 3 — THE CASCADE

One seed unfolds into everything: seed → title/engine/ABC → cast + motifs →
21-image matrix (7 image-types × past/present/future) → camera grammar →
prompts on every node, each editable, each regenerable, each STAGE-able.
Deterministic golden floor at every level; the model upgrades any node.

## Law 4 — NOTHING UNASKED, NOTHING SILENT

Compose plans; GENERATE builds — per node, per section, or all. Every failure
is loud (AI ERROR toast + AI·LOG line). Every stand-in is labeled. The scale,
frustum, and content of everything built is inspectable (VIEW/JUMP/focus).
